# 排班系统移动端设计文档

## 1. 项目概述

### 1.1 项目定位
- **项目名称**：排班系统移动端
- **目标平台**：Android WebView / 混合应用
- **技术栈**：Vue 3 + Vant 4
- **核心用户**：大学生值班人员

### 1.2 设计目标
1. **触控优先**：所有交互元素适合手指操作（最小 44px 触控区域）
2. **单手操作**：核心功能在屏幕下半部分可完成
3. **即时反馈**：操作后立即显示结果，减少页面跳转
4. **简洁高效**：常用功能 3 步内完成

---

## 2. 设计原则

### 2.1 视觉层级
- 重要信息优先展示
- 使用留白区分内容区块
- 色彩引导用户注意力

### 2.2 交互设计
- **手势操作**：滑动切换、长按编辑
- **底部操作**：主要按钮放置在底部，便于单手点击
- **即时编辑**：数据修改实时保存，减少确认步骤

### 2.3 内容密度
- 移动端采用"卡片式"布局
- 每屏显示 4-6 个主要元素
- 避免信息过载

成功:     #07C160 (绿色) - 无课状态、完成状态

## 9. 动效规范

### 9.1 过渡动画
| 场景 | 动画 | 时长 |
|------|------|------|
| 页面切换 | 滑动进入（从右到左）| 300ms |
| 返回上一页 | 滑动退出（从左到右）| 300ms |
| 弹窗出现 | 底部滑入 + 淡入 | 250ms |
| 弹窗消失 | 底部滑出 + 淡出 | 200ms |
| 状态切换 | 背景色渐变 | 200ms |
| 列表加载 | 骨架屏渐显 | 300ms |

### 9.2 反馈动画
| 场景 | 动画 |
|------|------|
| 点击按钮 | 缩放 0.95 |
| 切换状态 | 图标旋转 + 颜色渐变 |
| 保存成功 | Toast 提示 + 轻微震动（可选）|
| 下拉刷新 | 旋转箭头 + 文字提示 |
