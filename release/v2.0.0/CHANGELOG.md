## v2.0.0 更新公告

**统一接口格式**
- 后端响应统一使用 `utils.Success/Error`，前端拦截器直接提取 `data`，修复多处 `res.data` 访问错误。

**权限系统重构**
- 权限代码统一为冒号格式（如 `schedule:manage:dept`），移除废弃的下划线格式兼容代码。
- 授权/申请页面判断改为基于用户身份而非临时权限，拥有临时权限的用户仍可申请更多权限。

**审批流程优化**
- 全局权限申请自动路由到办公室管理员或系统管理员，不再发给无权审批的部门管理员。
- 部门管理员的待审批列表自动过滤全局权限申请，统计数字同步修正。
- 修复状态统计 bug（`string(int)` 转换错误），新增已拒绝统计卡片。

**功能修复**
- 新增修改密码接口与用户角色 RESTful 路由调整。
- 修复排班页面本部门分工不显示、异步加载顺序错误等问题。
- 值班安排页操作按钮改为"开发中"提示。
- 后端启动时默认设置北京时区。

**工程改进**
- 新增开发规范文档 `DEVELOPMENT_GUIDE.md`。
- `config.yaml` 加入 `.gitignore` 防止敏感信息泄露。
- Dockerfile 更新为使用 `frontend-v2`。
- 构建 Linux/Windows 多架构发行版（v2.0.0）。
